# Redaction and Publication Checklist

Use this checklist before publishing screenshots, PDFs, DOCX exports, policy tables, or generated images to this repository.

## Do Not Publish

| Item | Why It Should Not Be Public |
|---|---|
| Raw assignment PDFs | They may include course metadata, student identifiers, professor names, and grading context. |
| University cover pages | They contain identifying academic information and do not add portfolio value. |
| Raw resume file | It contains personal contact information and should not be inside a public project repository. |
| Full legal-analysis papers | Long raw papers look less professional than curated portfolio artifacts. |
| Screenshots with personal/course details | They can expose academic identifiers and submission metadata. |
| Full references pages | They are not needed for a recruiter-facing repo. |
| Legal claims stated as advice | The repo should remain educational and should not provide legal advice. |
| Sensitive incident details | Avoid private data, victim details, internal systems, or confidential evidence. |

## Safe to Publish

| Safer Artifact | Condition |
|---|---|
| Sanitized framework maps | Summarize laws/frameworks at a high level and include a disclaimer. |
| Case-study tables | Focus on governance lessons, not raw assignment answers. |
| Self-created diagrams | Use SVG/Mermaid diagrams without personal or course information. |
| Executive summaries | Keep them concise, policy-focused, and public-safe. |
| Redacted comparison tables | Crop tightly and remove any identifying academic context. |
| Interview talking points | Explain your analysis process and professional judgment. |

## Public Disclaimer to Keep

Use this language in the README and any LinkedIn/GitHub description:

> This is an academic/portfolio case study. It is not legal advice. It does not contain raw assignment PDFs, student identifiers, UIN, professor/course cover pages, private data, privileged information, or confidential legal analysis.

## Final Pre-Publish Review

Before pushing a file:

- [ ] No student ID, UIN, phone number, personal email, or professor name.
- [ ] No university cover page or course submission page.
- [ ] No raw assignment PDF or full paper.
- [ ] No legal advice wording.
- [ ] No personal data, private incident details, or confidential information.
- [ ] No copyrighted logos or brand assets unless clearly allowed.
- [ ] File adds evidence of skill: governance, privacy, policy mapping, incident response, compliance, or executive communication.

## Good GitHub Hygiene

- Use concise Markdown artifacts.
- Prefer diagrams and tables over long essays.
- Keep source documents private unless fully redacted.
- Use clear file names and folder structure.
- Maintain a professional tone suitable for recruiters and hiring managers.
